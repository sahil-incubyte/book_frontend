# Book Frontend

A book library web app built with **Next.js** and **Apollo Client**, talking to a
Ruby on Rails **GraphQL** backend. It supports full CRUD on books, an Apollo
normalized cache with optimistic updates, global client state via both React
Context and Redux Toolkit, and a component test suite with mocked GraphQL. The UI
is built with **Chakra UI v3** and **Tailwind CSS v4** — a custom theme, light/dark
mode, and a responsive layout.

This is the frontend companion to the `book_api` Rails GraphQL service.

## Features

- **Browse & view books** — list all books and view a single book's details.
- **Create / edit / delete** — with server-side validation errors surfaced in the UI.
- **Apollo cache management** — direct cache writes after create (no refetch) and
  **optimistic delete** (`cache.evict`) so the UI updates instantly.
- **Global state, two ways** — toast notifications via **React Context**, and a
  favorites feature via **Redux Toolkit** (a deliberate side-by-side comparison).
- **End-to-end type safety** — GraphQL operations are code-generated into typed
  documents, so query/mutation results and variables are fully typed.
- **Polished, themeable UI** — Chakra UI v3 components with a custom brand theme,
  **light/dark mode** (via `next-themes`), and responsive layouts. Tailwind CSS v4
  handles utility-level layout alongside it.
- **Tested** — Jest + React Testing Library with Apollo's `MockedProvider`
  (~99% statement coverage), no backend required to run tests.

## Prerequisites

- **Node.js 22** (pinned via [`.nvmrc`](.nvmrc)). With `nvm`: `nvm use`.
- The **`book_api` Rails backend** running locally (defaults to `http://localhost:3002`).

## Getting Started

1. Install dependencies:
   ```bash
   nvm use          # Node 22
   npm install
   ```

2. Configure the backend origin. Create `.env.local`:
   ```bash
   BOOK_API_URL=http://localhost:3002
   ```

3. Start the backend (in the `book_api` project):
   ```bash
   bin/rails server -p 3002
   ```

4. Start the frontend:
   ```bash
   npm run dev
   ```
   Open [http://localhost:3000/books](http://localhost:3000/books).

### How the frontend reaches GraphQL

The browser only ever calls the **same-origin** path `/graphql`. Next.js rewrites
that request to the Rails backend (see [`next.config.ts`](next.config.ts)), so there
is **no CORS** to configure. The backend address comes from `BOOK_API_URL`.

```
Browser ──POST /graphql──▶ Next.js (:3000) ──rewrite──▶ Rails (:3002)/graphql
```

## GraphQL Code Generation

GraphQL operations live in `lib/graphql/*.graphql`. Running codegen introspects the
live backend schema and generates fully-typed documents into `lib/gql/graphql.ts`.

```bash
nvm use            # Node 22 (codegen requires Node >= 20)
npm run codegen
```

Components import friendly names from the [`lib/graphql/books.ts`](lib/graphql/books.ts)
facade (e.g. `GET_BOOKS`, `CREATE_BOOK`) and pass them to Apollo hooks:

```ts
import { useQuery } from "@apollo/client/react";
import { GET_BOOKS } from "@/lib/graphql/books";

const { data, loading, error } = useQuery(GET_BOOKS); // data is fully typed
```

Re-run `npm run codegen` whenever a `.graphql` file or the backend schema changes.

## State Management

- **Server state** lives in the Apollo cache — never duplicated elsewhere.
- **Notifications** (toasts) use **React Context** — simple, low-frequency UI state.
- **Favorites** use **Redux Toolkit** — a slice with typed selectors/actions,
  demonstrating when the extra structure (and Redux DevTools) is worth it.

## Styling & Theming

The app deliberately combines two styling systems, each in its own lane:

- **Chakra UI v3** owns components, theming, and color mode. The custom theme lives
  in [`components/ui/system.ts`](components/ui/system.ts) (`createSystem` with a
  `brand` palette + Geist fonts). Components style via props and semantic tokens
  (`bg.subtle`, `fg.muted`), so they adapt to light/dark automatically.
- **Tailwind CSS v4** handles utility-level layout (flex, spacing, responsive
  prefixes). Config is CSS-based in [`app/globals.css`](app/globals.css) — there is
  no `tailwind.config.js`.

Two integration details make them coexist cleanly:

1. **One reset.** Chakra owns the CSS reset; Tailwind's Preflight is omitted
   (`globals.css` imports only Tailwind's `theme` + `utilities` layers) to avoid a
   double reset.
2. **One dark-mode switch.** `next-themes` toggles a `.dark` class on `<html>`.
   Chakra tokens follow it, and a `@custom-variant dark` rule points Tailwind's
   `dark:` variant at the same class — so one click flips both systems together.

The color-mode toggle button lives in the Books page header.

## Testing

```bash
npm test              # run once
npm run test:watch    # watch mode
npm run test:coverage # with coverage report
```

Tests never hit the real backend — GraphQL is mocked with Apollo's
`MockedProvider` via the `renderWithProviders` helper. Coverage sits around 99%
of statements.
